Flash Memory Cards

先记住这些 one-liner，再展开详细内容。

Activation

ReLU is king

$\max(0,x)$: 不饱和、计算快、收敛快。Dead ReLU 是唯一缺陷。

Backprop

Chain Rule 一切

Add gate 复制梯度，Multiply gate 交换缩放，Max gate 路由给赢家。

CNN

Output Size

$\frac{W - F + 2P}{S} + 1$ 背下来，所有CNN计算题都需要。

LSTM

Forget-Input-Output

三个 sigmoid 门 + tanh 候选值。Cell state 做加法更新，梯度更容易流动。

Training

BN: Train ≠ Test

训练用 batch 统计量，推理用 running average。BN 推理时不关掉。

GAN

Minimax Game

G 想 $D(G(z))→1$, D 想 $D(x)→1$ 且 $D(G(z))→0$。Mode collapse 是主要问题。

VAE

ELBO = Recon + KL

Reparameterization trick: $z = \mu + \sigma \cdot \epsilon$。优化 ELBO 来逼近 marginal likelihood。

GNN

Message Passing

节点聚合邻居信息更新自身。Sum/Mean/Max 聚合保证排列不变性。

Init

Xavier → Tanh, He → ReLU

零初始化 = 对称性不破 = 所有神经元学一样的东西。

Linear Classifier

$$f(\mathbf{x}) = W\mathbf{x} + \mathbf{b}$$ 输入 $\mathbf{x} \in \mathbb{R}^D$, 权重 $W \in \mathbb{R}^{C \times D}$, 偏置 $\mathbf{b} \in \mathbb{R}^C$, 输出 $C$ 个类的分数。
注意: 堆叠线性层（不加非线性）仍然等价于一个线性层: $W_2(W_1\mathbf{x}) = W'\mathbf{x}$

Bias-Variance Tradeoff

Underfitting (High Bias)

训练集和测试集都差

模型太简单，学不到数据规律。
解决: 增大模型容量、减少正则化、更多特征。

Overfitting (High Variance)

训练集好、测试集差

模型太复杂，记住了训练数据噪声。
解决: 更多数据、正则化、dropout、early stopping、数据增强。

$$\text{Total Error} = \text{Bias}^2 + \text{Variance} + \text{Irreducible Noise}$$
考试高频题型: 给定 "training error 低 validation error 高" → overfitting；两者都高 → underfitting。问你应该怎么修 — 分别对应增正则化 vs 增容量。

Loss Functions

Loss 衡量模型预测与真实值的差距。训练 = 最小化 loss。

Loss公式用途特点
MSE $L = \frac{1}{n}\sum(y_i - \hat{y}_i)^2$ 回归 可导，对大误差惩罚重
Cross-Entropy $L = -\sum_{c=1}^C y_c \log p_c$ 多分类 (配合 Softmax) 对低概率正确类惩罚极强
Binary CE $L = -[y\log\sigma(z)+(1-y)\log(1-\sigma(z))]$ 二分类 配合 sigmoid 输出
Hinge Loss $L = \max(0, 1-y\cdot f(x))$ SVM margin 分类 鼓励 margin 分离

Softmax + Cross-Entropy 完整链

Softmax: $$p_j = \frac{e^{s_j}}{\sum_k e^{s_k}}$$ Cross-Entropy: $$L = -\log p_{\text{correct}} = -s_{\text{correct}} + \log\!\Bigl(\sum_k e^{s_k}\Bigr)$$ 梯度（非常简洁）: $$\frac{\partial L}{\partial s_j} = p_j - y_j$$ 即 "预测概率减去真实标签 (one-hot)"
初始 Loss 验证: 对 $K$ 分类，随机初始化时每类概率 $\approx 1/K$，初始 loss $\approx \log K$。
例如 10 分类: 初始 loss $\approx \log 10 \approx 2.3$。如果你的初始 loss 偏离这个值很多，说明实现有 bug。

Activation Functions 激活函数

非线性是让 NN 超越线性模型的关键。没有非线性激活，再深的网络也只是一个线性变换。

激活公式范围导数优点缺点
Sigmoid $\sigma(x)=\frac{1}{1+e^{-x}}$ $(0,1)$ $\sigma(1-\sigma)$ 输出可解释为概率 饱和梯度消失、非零中心、exp 计算较贵
Tanh $\tanh(x)$ $(-1,1)$ $1-\tanh^2(x)$ 零中心 饱和区仍有梯度消失
ReLU $\max(0,x)$ $[0,\infty)$ $x>0$: 1, else 0 计算简单、收敛快6x、不正区间饱和 Dead ReLU、非零中心
Leaky ReLU $\max(\alpha x, x)$ $(-\infty,\infty)$ $x>0$: 1, else $\alpha$ 解决 Dead ReLU 多一个超参 $\alpha$
PReLU 同 Leaky, $\alpha$ 可学习 $(-\infty,\infty)$ 同上 $\alpha$ 自动调整 增加参数量
GELU $x \cdot \Phi(x)$ $\approx(-0.17,\infty)$ 平滑，非分段 Transformer 标配 计算复杂

Key Fact #1

为什么 ReLU 成为现代标准

1. 计算极简（一个 max）；2. 正区间梯度恒为 1，不会饱和梯度消失。AlexNet 用 ReLU 替换 sigmoid/tanh 是深度学习史关键转折。

Key Fact #2

Dead ReLU Problem

如果 ReLU 神经元输入永远为负（大 LR 或差的初始化），永远输出 0、梯度 0，再也学不回来。Leaky/PReLU 就是为此设计。

Sigmoid 数值考题: $\sigma(0)=0.5$, $\sigma'(0)=0.25$, $\sigma(z)$ 在 $z>0$ 时 $>0.5$, 在 $z<0$ 时 $<0.5$。这类数值判断经常出现在 multi-select 中。

Backpropagation 反向传播

利用 Chain Rule 从 loss 逆向高效计算每个参数的梯度。

Computational Graph & Backprop Flow

→
$z = Wx+b$
→
$a = \sigma(z)$
→
Loss $L$
← Backward: $\frac{\partial L}{\partial W} = \frac{\partial L}{\partial a} \cdot \frac{\partial a}{\partial z} \cdot \frac{\partial z}{\partial W}$
Chain Rule: $$\frac{dy}{dx} = \frac{df}{dg} \cdot \frac{dg}{dx} \quad \text{if } y = f(g(x))$$ Fully Connected Layer Backward:
设 $z = Wx + b$, $a = \sigma(z)$, $\delta = \frac{\partial L}{\partial z} = \frac{\partial L}{\partial a} \cdot \sigma'(z)$ $$\frac{\partial L}{\partial W} = \delta \cdot x^\top, \quad \frac{\partial L}{\partial b} = \delta, \quad \frac{\partial L}{\partial x} = W^\top \delta$$ Vectorized (batch): $Y = XW + b$, $X \in \mathbb{R}^{N \times D}$, $W \in \mathbb{R}^{D \times M}$ $$dW = X^\top \, dY, \quad dX = dY \, W^\top, \quad db = \sum_{\text{batch}} dY$$

GateForwardLocal Gradient直觉

Vanishing Gradient

梯度消失问题

深层网络中如果每层梯度 $< 1$（如 sigmoid 饱和区），连乘后指数衰减，前面的层几乎收不到梯度。
解决: ReLU、ResNet、BatchNorm。

Exploding Gradient

梯度爆炸问题

每层梯度 $> 1$ 时连乘后指数增长，参数更新过大导致发散。
解决: Gradient clipping, 更好的初始化。

为什么要缓存前向值 (Forward Cache): Add gate 不需要缓存（导数是常数），但 Multiply gate、max gate、sigmoid 等都需要前向输入或输出来计算局部梯度。

为什么 CNN 适合图像

每个 filter 只看一小块区域 (receptive field)，天然符合图像的局部特征结构（边缘、纹理、角点）。

卷积层详细过程

Convolution Operation

Convolution Math & Parameter Counting

Output Size Formula: $$\text{Output} = \frac{W - F + 2P}{S} + 1$$ $W$ = input width/height, $F$ = filter size, $P$ = padding, $S$ = stride

Worked Examples (高频考题类型)

ProblemSetupAnswer
Example 1 Input: $32\times32\times3$, Filter: $5\times5$, 6 filters, S=1, P=0 Output: $(32-5+0)/1+1 = 28$, so $28\times28\times6$
Example 2 Input: $4\times6\times3$, Output: $3\times4\times10$ 10 filters, each depth 3. 可验证: $2\times2$ filter, S=2, P=1
Bonus Quiz Input W=9, F=3, S=2, desired output W=9 $(9-3+2P)/2+1=9 \Rightarrow P=5$

Parameter Counting

每个 filter 的参数: $F \times F \times C_{\text{in}}$ weights $+ 1$ bias $= F^2 C_{\text{in}} + 1$
总参数: $K \times (F^2 C_{\text{in}} + 1)$

Example: 10 个 $5\times5$ filter, 3-channel input
每个 filter: $5\times5\times3 = 75$ weights $+ 1$ bias $= 76$
总参数: $10 \times 76 = 760$

Pooling Layer

Max Pooling

取区域最大值

减小空间分辨率，无可学习参数。反向传播时梯度只流向最大值位置。

Avg Pooling

取区域平均值

同样无可学习参数。梯度均匀分配给区域内所有位置。

Padding 的作用: 1) 保持空间大小 (when S=1, P=(F-1)/2); 2) 保留边界信息不被丢弃。Padding 不改变 filter 数量。

Residual Connections (ResNet)

Residual Block Architecture

$$\mathbf{y} = \mathcal{F}(\mathbf{x}, \{W_i\}) + \mathbf{x}$$ Block 只需要学习 残差 $\mathcal{F}(\mathbf{x})$（偏离 identity 的部分），而不是从头学整个映射。
当最优映射接近 identity 时，残差接近 0，比从头学 identity 更容易。
为什么有效

Three Benefits

1. 信息高速公路: 输入直接到输出，信息不被层层扭曲。
2. 梯度高速公路: 梯度通过 skip 直接传到浅层。
3. 学残差更容易: 学"什么要改"比学"完整映射"简单。

注意

ResNet 不移除非线性

Skip connection 是加法捷径，并不取消了每个 block 里的非线性激活。ReLU 仍然存在于 residual branch 中。

隐藏状态 $h_t$ 在相邻时间步之间传递 · 所有时间步共享权重 $W$
Vanilla RNN: $$h_t = \tanh(W_{hh}\,h_{t-1} + W_{xh}\,x_t + b)$$ $$y_t = W_{hy}\,h_t$$

RNN Parameter Counting

设 input dim $= d_x$, hidden dim $= d_h$:
$$W_{hh}: d_h \times d_h = d_h^2 \text{ params}$$ $$W_{xh}: d_h \times d_x \text{ params}$$ $$b: d_h \text{ params}$$ Example: $d_x=10, d_h=20$: total $= 400 + 200 + 20 = 620$ params (不是 420！)
Weight Sharing

增加 T 不增加参数

序列长度 T 增加只增加计算量和 BPTT 路径长度，不改变可训练参数数量。同一个 cell 重复使用。

BPTT

Backpropagation Through Time

展开整个序列计算 loss，然后沿时间反向传播梯度。Truncated BPTT: 只反向传播有限步数，降低计算开销。

Vanilla RNN 的核心问题: 反向传播时 $\frac{\partial h_t}{\partial h_{t-1}}$ 包含 $W_{hh}^\top$ 的连乘。Largest singular value > 1 → 梯度爆炸; < 1 → 梯度消失。这就是为什么需要 LSTM。

Long Short-Term Memory (LSTM)

通过引入 cell state 和三个 gate 来解决 vanilla RNN 的梯度消失问题，能学习长距离依赖。

LSTM Cell Internal Structure

LSTM 完整公式: $$f_t = \sigma\bigl(W_f [h_{t-1}, x_t] + b_f\bigr) \quad \text{(Forget Gate — sigmoid)}$$ $$i_t = \sigma\bigl(W_i [h_{t-1}, x_t] + b_i\bigr) \quad \text{(Input Gate — sigmoid)}$$ $$\tilde{c}_t = \tanh\bigl(W_c [h_{t-1}, x_t] + b_c\bigr) \quad \text{(Candidate — tanh)}$$ $$c_t = f_t \odot c_{t-1} + i_t \odot \tilde{c}_t \quad \text{(Cell State Update — 加法!)}$$ $$o_t = \sigma\bigl(W_o [h_{t-1}, x_t] + b_o\bigr) \quad \text{(Output Gate — sigmoid)}$$ $$h_t = o_t \odot \tanh(c_t) \quad \text{(Hidden State Output)}$$

Embedding Layer

From one-hot to dense

V=10000 词表 → $10000 \times 300$ 矩阵。每个 token 映射到 300 维稠密向量。比 one-hot 高效得多。矩阵乘 one-hot = 取出一列。

Autoregressive Generation

逐 token 生成

前面的输出 feed 回来作为下一步输入。未来 token 在生成时不可见 → Bidirectional RNN 不适合标准自回归生成。

Weight Initialization

方法方差适用激活为什么
Zero Init 0 ❌ 不要用 所有神经元对称 → 梯度相同 → 学到同样的东西
Small Random ~0.01 浅层可用 深层会衰减到 0 (activations → 0)
Xavier $\text{Var}(w) = \frac{1}{D_{\text{in}}}$ Tanh, Sigmoid 保持输入输出方差一致
He (Kaiming) $\text{Var}(w) = \frac{2}{D_{\text{in}}}$ ReLU ReLU 丢弃一半信号（负半轴），所以要 ×2 补偿
Xavier 推导直觉:
设 $y = \sum_{i=1}^{D_{\text{in}}} x_i w_i$, 要求 $\text{Var}(y) = \text{Var}(x_i)$
$$\text{Var}(y) = D_{\text{in}} \cdot \text{Var}(x_i) \cdot \text{Var}(w_i)$$ 令 $\text{Var}(w_i) = \frac{1}{D_{\text{in}}}$ 即可保持方差不变
考试高频: 零初始化为什么不行？→ "对称性不破" (symmetry not broken) → 所有 neurons 收到相同梯度 → 等于一个神经元。

Batch Normalization

Training: $$\hat{x}_i = \frac{x_i - \mu_B}{\sqrt{\sigma_B^2 + \varepsilon}}$$ $$y_i = \gamma \hat{x}_i + \beta$$ 其中 $\mu_B, \sigma_B^2$ 是当前 mini-batch 的均值和方差，$\gamma, \beta$ 是可学习参数

Inference: 使用训练时累积的 running mean 和 running variance（不用当前 batch 统计量）
BN 的好处

Multiple Benefits

1. 改善梯度流 (让梯度不那么依赖 scale)
2. 允许更大学习率
3. 减少对初始化的敏感度
4. 有轻微正则化效果

Train vs Test 区别

⚠️ 考试高频考点

Training: 用 mini-batch 的 $\mu_B, \sigma_B^2$
Test: 用训练时累积的 running statistics
BN 推理时并不关掉，只是统计量来源不同。

BN 位置: 通常放在 Conv/FC 之后、非线性激活之前。即: Conv → BN → ReLU。

Regularization & Dropout

L2 Regularization (Weight Decay)

$L_{\text{total}} = L + \lambda \|W\|^2$

惩罚大权重，鼓励模型使用更小更分散的权重。防止过拟合。

L1 Regularization

$L_{\text{total}} = L + \lambda \|W\|_1$

鼓励稀疏权重（很多权重趋向 0）。可用于特征选择。

Dropout

Dropout: Training vs Test

Training (p=0.5)
Test (全部使用)
Standard Dropout: Training 时随机将 neuron 输出置 0 (概率 $p$)，Test 时全部使用但乘以 $(1-p)$

Inverted Dropout (更常用): Training 时 drop 后立即除以 $(1-p)$ 补偿，Test 时不需要任何额外 scaling
Inverted Dropout 是考试重点: Training scale up (÷ (1-p))，Test 不改变。Test 时不再 drop units。

Optimizers

SGD: $$\theta_{t+1} = \theta_t - \eta \nabla L(\theta_t)$$ SGD + Momentum: ($\beta$ 通常 0.9) $$v_t = \beta v_{t-1} + \nabla L(\theta_t)$$ $$\theta_{t+1} = \theta_t - \eta v_t$$ Adam: $$m_t = \beta_1 m_{t-1} + (1-\beta_1)\nabla L \quad \text{(一阶矩)}$$ $$v_t = \beta_2 v_{t-1} + (1-\beta_2)(\nabla L)^2 \quad \text{(二阶矩)}$$ $$\hat{m}_t = \frac{m_t}{1-\beta_1^t}, \quad \hat{v}_t = \frac{v_t}{1-\beta_2^t} \quad \text{(偏差修正)}$$ $$\theta_{t+1} = \theta_t - \frac{\eta \hat{m}_t}{\sqrt{\hat{v}_t}+\varepsilon}$$ 常见默认: $\beta_1=0.9, \beta_2=0.999, \varepsilon=10^{-8}$
OptimizerKey FeatureProsCons
SGD最简单直接、理论性质好震荡大、收敛慢
Momentum累积历史梯度方向减少震荡、加速收敛多一个超参
AdaGrad累积 $g^2$，自适应 LR稀疏梯度友好LR 只降不升
RMSProp滑动平均代替累积解决 AdaGrad LR 衰减仍需调参

Adam Bias Correction

为什么需要偏差修正？

$m,v$ 初始化为 0 → 前几步估计值偏低。除以 $(1-\beta^t)$ 补偿初始偏差。注意: 这不是 gradient clipping！

Learning Rate Warm-up

前几百步线性增大 $\eta$

防止初始阶段大梯度导致不稳定更新。之后再用 cosine annealing 或 step decay 衰减。不保证立即收敛。

Gradient Clipping: 主要用于防止梯度爆炸（特别是 RNN），不能修复梯度消失。当 $\|\nabla L\| > \text{threshold}$ 时缩放梯度。

Generative Adversarial Networks (GAN)

通过 Generator 和 Discriminator 的对抗博弈，学习从噪声生成真实数据。

GAN Architecture

Minimax 目标函数: $$\min_G \max_D \; V(D,G) = \mathbb{E}_{x \sim p_{\text{data}}}[\log D(x)] + \mathbb{E}_{z \sim p_z}[\log(1-D(G(z)))]$$ Discriminator (最大化): 让 $D(x) \to 1$ (真的判真)，$D(G(z)) \to 0$ (假的判假)
Generator (最小化): 让 $D(G(z)) \to 1$ (假的骗过 D)
Non-Saturating Loss

为什么改目标函数

原始 G 目标 $\min \log(1-D(G(z)))$：当 D 很强时梯度趋 0 (饱和)。
改为 $\max \log D(G(z))$：提供更强的梯度信号。保持相同的均衡思想。

Mode Collapse

GAN 最大的问题

Generator 只生成少数几种能骗过 D 的样本，多样性丧失。生成的图像高度重复。

Conditional GAN (cGAN): Generator 输入不仅是噪声 $z$，还包括条件信息（如类别标签、图像），从而控制生成内容。例: pix2pix 用图像作条件做图像翻译。

Variational Autoencoder (VAE)

通过约束隐空间为概率分布来实现生成能力。先学 Autoencoder，再理解 VAE。

Reparameterization Trick: $z = \mu + \sigma \odot \epsilon, \; \epsilon \sim \mathcal{N}(0,I)$
Autoencoder vs VAE:
Autoencoder: 学确定性映射 $z = f(x)$, $\hat{x} = g(z)$ — 不能采样生成新数据
VAE: 学概率映射 — 隐空间有结构（Gaussian prior），可以采样 $z \sim \mathcal{N}(0,I)$ 生成新数据

ELBO (Evidence Lower Bound): $$\log p_\theta(x) \geq \underbrace{\mathbb{E}_{q_\phi(z|x)}[\log p_\theta(x|z)]}_{\text{Reconstruction}} - \underbrace{D_{\mathrm{KL}}\bigl(q_\phi(z|x) \;\|\; p(z)\bigr)}_{\text{KL Regularization}}$$ Reparameterization Trick: $$z = \mu + \sigma \odot \epsilon, \quad \epsilon \sim \mathcal{N}(0, I)$$ 将随机采样转化为确定性函数 + 外部噪声，使得梯度可以通过 $\mu, \sigma$ 反向传播
Reconstruction Loss

重建输入数据

$\mathbb{E}[\log p_\theta(x|z)]$: 让 decoder 输出尽可能接近输入。保证 encoder 学到有用的信息。

KL Divergence Loss

正则化隐空间

$D_\mathrm{KL}(q_\phi(z|x) \| p(z))$: 让 encoder 输出的分布接近标准正态。使得隐空间连续、有意义。去掉 KL 就失去了正则化。

Generation: 训练好后，从 $z \sim \mathcal{N}(0,I)$ 采样，通过 Decoder 即可生成新数据。不需要 Encoder。

Graph Neural Networks (GNN)

将神经网络扩展到图结构数据。通过 message passing 在节点之间传递和聚合信息。

Message Passing Framework

Message Passing Update Rule: $$h_v^{(l+1)} = \text{UPDATE}\!\Bigl(h_v^{(l)}, \; \text{AGGREGATE}\bigl(\{h_u^{(l)} : u \in \mathcal{N}(v)\}\bigr)\Bigr)$$ 常见 Aggregation: Sum, Mean, Max — 都是对称函数，保证排列不变/等变

GCN 层 (简化版): $$H^{(l+1)} = \sigma\!\bigl(\hat{A} \, H^{(l)} \, W^{(l)}\bigr)$$ 其中 $\hat{A}$ 是归一化邻接矩阵, $W^{(l)}$ 是可学习权重

GNN Task Types

Node Classification

节点分类

预测每个节点的标签。例: 社交网络中用户标签预测。利用部分已标注节点 (semi-supervised)。

Link Prediction

链接预测

预测两个节点之间是否应该存在边。例: 推荐系统中预测用户-商品交互。

Graph Classification

图分类

预测整张图的标签。需要 Readout/Pooling 将所有节点表示聚合为图级别表示。

Permutation Invariance/Equivariance: GNN 的输出不能因为节点编号不同而改变。使用对称聚合函数 (sum, mean, max) 是满足这一性质的关键。

Exam Tips & Sanity Checks

Sanity Check #1

初始 Loss 是否合理

$K$ 分类初始 loss $\approx \log K$
10 classes → ~2.3
如果偏差很大，实现有 bug。

Sanity Check #2

能否在小数据集上过拟合

取 5-10 个样本训练，loss 应该能降到接近 0。
如果不能 → 实现或架构有问题。

Sanity Check #3

监控 Validation 而非只看 Training

Training loss 下降不代表模型正确。
要看 validation loss/accuracy 确认泛化能力。
