Show Solution
Batch GD: Uses all N samples per update. Advantage: accurate gradient estimate, stable convergence.
SGD: Uses 1 random sample per update. Advantage: very fast update steps, can escape local minima due to noise.

Question 2 — Convolutional Neural Networks

20 Points
(a)6 pts
List three distinct advantages of CNNs over fully-connected (dense) networks for image processing tasks. For each, give a one-sentence explanation.
Show Solution
1. Parameter Sharing: The same kernel (filter) is applied at every spatial position, drastically reducing the number of learnable parameters compared to a fully-connected layer.
2. Local Connectivity (Sparse Interactions): Each output neuron depends only on a small local receptive field, not the entire input — reflecting the local structure of natural images and reducing computation.
3. Translation Equivariance: A CNN can detect the same feature (e.g., an edge) regardless of where it appears in the image, because the same filters are scanned across all positions.

Question 4 — Flynn's Taxonomy, SIMD, Processor Visualization & Memory Banks

40 Points
(a)8 pts
Classify each of the following into Flynn's taxonomy: SISD, SIMD, MISD, or MIMD.
#DescriptionClassification
(i)A single-core CPU executing a sequential scalar program (no vectorization).
(ii)A GPU warp: 32 threads execute an identical instruction simultaneously on different data elements.
(iii)Two CPU cores each running a completely different application simultaneously.
(iv)Google's TPU v1 systolic array: data flows through many PEs, each performing a different stage of the computation.
Show Solution
(i) SISD — Single Instruction, Single Data: the classic sequential CPU model.
(ii) SIMD — Single Instruction, Multiple Data: one instruction broadcasts across 32 lanes simultaneously. This is the core of GPU parallelism (also called SIMT in hardware).
(iii) MIMD — Multiple Instruction, Multiple Data: each core independently fetches and executes its own instruction stream on its own data.
(iv) MISD — Multiple Instruction, Single Data: the same activation value flows through all PEs in a row; each PE applies a different weight (a different "instruction"/operation). Data stream is reused; instructions differ per PE.

The task is: compute $C[i] = A[i] + B[i]$ for $i = 0, 1, 2, 3$ (4 elements).

Show Solution
Scalar (total = 12 cycles): One PE processes elements sequentially. E0 takes cycles 1–3, E1 takes cycles 4–6, E2 takes 7–9, E3 takes 10–12. Total = $4 \times 3 = \mathbf{12}$ cycles. PE is idle 0% (fully busy but no parallelism).
Array (total = 3 cycles): Four dedicated PEs all start simultaneously. All elements load in cycle 1, add in cycle 2, store in cycle 3. Total = 3 cycles. Speedup vs Scalar = $12/3 = 4\times$ — perfect linear scaling!
Vector Pipeline (total = 6 cycles): One pipelined unit with 3 stages. E0 enters at cycle 1. E1 enters at cycle 2 (stage 1 is free). After filling the pipeline at cycle 3, all 3 stages are active. Last element (E3) finishes at cycle 6. Formula: $\text{stages} + (N-1) = 3 + 3 = \mathbf{6}$ cycles. Speedup vs Scalar = $12/6 = 2\times$.
Key insight: Array = spatial parallelism (multiple PEs at once). Vector = temporal parallelism (pipeline overlap in time). Array is faster here but needs 4× the hardware.
