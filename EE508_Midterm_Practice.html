Q1 Gradient Descent, Loss Function & Chain Rule 35 Points

Setup

The following figure shows a two-layer neural network with no bias terms:

x1=1 --(w1)--+ +--[ z = w1*x1 + w2*x2 ]--ReLU--[ a ]--(v)--[ yhat = v*a ] x2=2 --(w2)--+ Initial weights: w1=2, w2=1, v=3 True label: y=10 Loss: L=(yhat-y)^2 Learning rate: alpha=0.05
Part 1.1 — Forward Pass  (5 pts)

Compute the pre-activation $z$ and hidden activation $a = \text{ReLU}(z)$.

$z = w_1 x_1 + w_2 x_2 = 2(1)+1(2) = \mathbf{4}$

$a = \text{ReLU}(4) = \mathbf{4}$  (slope = 1 since $z>0$)

Part 1.2 — Output & Loss  (5 pts)

Compute $\hat{y}$ and loss $L$.

$\hat{y}=v\cdot a = 3\times4 = \mathbf{12}$

$L=(\hat{y}-y)^2=(12-10)^2=\mathbf{4}$

Part 1.3 — Chain Rule for $w_1$  (10 pts)

Using the chain rule, compute $\dfrac{\partial L}{\partial w_1}$. Fill in each factor individually, then the combined answer.

$\dfrac{\partial L}{\partial w_1} = \underbrace{\dfrac{\partial L}{\partial\hat{y}}}_{(1)} \cdot\underbrace{\dfrac{\partial\hat{y}}{\partial a}}_{(2)} \cdot\underbrace{\dfrac{\partial a}{\partial z}}_{(3)} \cdot\underbrace{\dfrac{\partial z}{\partial w_1}}_{(4)}$





(1) $2(\hat{y}-y)=2(12-10)=\mathbf{4}$

(2) $\hat{y}/\partial a = v = \mathbf{3}$

(3) ReLU derivative at $z=4>0$ is $\mathbf{1}$

(4) $\partial z/\partial w_1 = x_1 = \mathbf{1}$

dL/dw1 = 4 x 3 x 1 x 1 = 12
Part 1.4 — Chain Rule for $v$ and $w_2$  (10 pts)

(a) Compute $\partial L/\partial v$.   (b) Compute $\partial L/\partial w_2$.

(a) $\partial L/\partial v = \partial L/\partial\hat{y} \cdot \partial\hat{y}/\partial v = 4 \times a = 4\times4 = \mathbf{16}$

(b) Same chain as 1.3 but $\partial z/\partial w_2 = x_2 = 2$:

dL/dw2 = 4 x 3 x 1 x 2 = 24
Part 1.5 — Gradient Descent Update  (5 pts)

Perform one GD step on $w_1$ and $v$ ($\alpha=0.05$).

$w_1^\text{new} = 2 - 0.05\times12 = \mathbf{1.4}$

$v^\text{new} = 3 - 0.05\times16 = \mathbf{2.2}$

Part 1.6 — SGD vs. Full-Batch GD  (Conceptual)

Suppose the dataset has 10,000 samples and we switch from full-batch GD to SGD (batch size=1).

(a) Does each SGD step give the exact same gradient as a full-batch step? Why or why not?

(b) Which typically converges faster in wall-clock time, and why?

(a) No. SGD uses a single random sample, producing a noisy estimate of the true gradient. They match only in expectation.

(b) SGD is typically faster in wall-clock time: each iteration costs 1/N of the compute, enabling far more parameter updates per second. The high update frequency usually dominates the noise disadvantage for large datasets.

Part 2.2 — Same Padding Formula  (5 pts)

For Out=In with stride S=1: derive P in terms of F. What value of P is needed for F=5?

$In=(In-F+2P)+1 \Rightarrow 0=-F+2P+1 \Rightarrow P=(F-1)/2$

For F=5: $P=(5-1)/2=\mathbf{2}$

Part 2.3 — 2D Multi-Channel Output  (5 pts)
  • Input: $(C,H,W)=(3,5,5)$
  • Conv layer: 8 filters, each $(C,K_H,K_W)=(3,3,3)$, $S=1$, $P=0$

What are the output dimensions $(H_\text{out}, W_\text{out}, D_\text{out})$?

$H_\text{out}=(5-3)/1+1=\mathbf{3}$,  $W_\text{out}=\mathbf{3}$,  $D_\text{out}=\text{num filters}=\mathbf{8}$

Note: filter depth (=3) must match input channels. Output depth = number of filters, not filter depth.

Part 2.4 — Convolution vs. Cross-Correlation  (5 pts)

What is the mathematical difference between convolution and cross-correlation? Which do CNNs actually compute? Does this distinction matter for learning?

Convolution flips (180°) the kernel before sliding. Cross-correlation does not flip.

CNNs technically compute cross-correlation, called “convolution” by convention.

No practical difference for learning: since weights are learned, the network simply learns the “flipped” version automatically.

Q3 Roofline Model — Arithmetic Intensity 20 Points

Hardware Specifications

  • Peak compute: 200 GFLOP/s
  • Peak memory bandwidth: 50 GB/s
  • Data type: FP32 (4 bytes per element)

We analyze square matrix multiply $C = A \times B$ where $A, B, C$ are each $N\times N$ FP32 matrices.

Part 3.1 — Ridge Point  (4 pts)

Compute the ridge point: $\text{Ridge} = \text{Peak Compute} / \text{Peak BW}$.

200 GFLOP/s / 50 GB/s = 4 FLOP/Byte

AI > 4 ⇒ compute-bound. AI < 4 ⇒ memory-bound.

Part 3.2 — Arithmetic Intensity for $N=200$  (10 pts)

Calculate the AI for $N=200$. Assume all data loaded from DRAM once. Show: FLOPs → Bytes → AI    (Hint: FLOPs=$2N^3$, Bytes=$3N^2\times4$)

FLOPs = 2 x 200^3 = 2 x 8,000,000 = 1.6 x 10^7
Bytes = 3 x 200^2 x 4 = 3 x 40,000 x 4 = 4.8 x 10^5
AI = 1.6e7 / 4.8e5 = 33.3 FLOP/Byte

Shortcut: $AI \approx N/6 = 200/6 \approx 33.3$ ✓

Part 3.3 — Bound Classification & Achievable Performance  (4 pts)

Is the $N=200$ GEMM memory-bound or compute-bound? What does the Roofline model predict for achievable performance?

AI=33.3 > Ridge=4 ⇒ Compute-bound. Achievable = peak = 200 GFLOP/s.

Part 3.4 — Effect of Doubling $N$  (2 pts)

If $N$ doubles to 400, what happens to AI and achievable performance? Justify.

New AI $\approx 400/6 \approx 66.7$ FLOP/B. Still >> ridge (4). Still compute-bound.

Achievable performance still = 200 GFLOP/s. Doubling N does not help — must upgrade hardware.

Q4 Flynn's Taxonomy, Scalar vs. Vector Processors & Memory Banks 40 Points

Setup — The Following Loop Runs on Different Processor Architectures

for (i = 0; i < 50; i++)
    C[i] = (A[i] + B[i]) / 2;

Latencies: LD=11  |  ADD=4  |  SHR (÷2)=1  |  ST=11  |  DECBNZ=2 cycles
Vector proc: 16 memory banks, bank latency=11 cycles, elements stored stride-1. Init overhead=2 cycles.

Part 4.1 — Flynn's Taxonomy  (8 pts)

Classify each system as SISD, SIMD, MISD, or MIMD.

Single-core scalar CPU (e.g., Intel Core i7)
GPU warp (32 threads, one instruction broadcast)
CRAY-1 vector processor
Multi-socket server (4 CPUs, each 8 cores)

SISD — single core, one instruction, one data stream.

SIMD — GPU warp: one instruction broadcast to 32 lanes, different data.

SIMD — CRAY-1 vector: one vector instruction operates on entire array.

MIMD — multi-socket: each core runs independent program on independent data.

Part 4.4 — Memory Bank Analysis  (10 pts)

16 banks, bank latency=11 cycles. $A[k]$ maps to bank $k\bmod16$.

(a) Stride=1. First access to bank 0 is at cycle 1. When is the next? Is there a conflict?

Next access to bank 0 at cycle:

Stride-1 uses 16 consecutive banks. Return to bank 0 after 16 accesses = cycle 17.

Bank 0 finishes at cycle 12. Since 17 > 12 ⇒ no conflict. ✓

16 banks ≥ 11-cycle latency: always enough gap between same-bank accesses.

(b) Stride=4. Accesses: A[0],A[4],A[8],A[12],A[16],... How many unique banks? Gap between same-bank accesses? Stall penalty?

Banks: 0,4,8,12,0,4,... ⇒ GCD(4,16)=4, only 4 unique banks.

Return to bank 0 every 4 accesses = gap of 4 cycles.

Stall = bank latency - gap = 11 - 4 = 7 cycles per re-access

Every 4th element causes a 7-cycle stall, drastically reducing throughput.
