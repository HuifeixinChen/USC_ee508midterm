Dataflow 和 Stationariness

这是硬件部分的真正核心。因为数据搬运的代价，往往比一次 MAC 更贵。

类型 固定什么 优点 缺点
Weight Stationary 权重 权重复用高，适合减少权重读取 大模型时不容易放下
Output Stationary partial sums 减少中间结果写回 其他数据可能搬得更多
Input Stationary 输入激活 输入复用高时有效 需要更复杂控制保证权重到位
统一理解

先记原则，再记名字

Stationary dataflow 的目标是让某类数据尽量留在近处，通过 reuse 减少昂贵的 memory access。

Eyeriss

Eyeriss is an energy-efficient CNN accelerator. The core idea is not just to add more MAC units, but to reduce expensive data movement. Eyeriss 是一个以能效为核心的 CNN 加速器，重点不是盲目堆算力，而是尽量少搬数据。

What It Is

Energy-first CNN accelerator

Eyeriss is built for convolution-heavy workloads where reuse matters. 它面向卷积层，设计目标是把可复用的数据尽量留在片上。

Why It Matters

DRAM access dominates energy

Moving data from off-chip DRAM can cost far more energy than doing the MAC itself. 从 DRAM 取数往往比乘加本身贵得多，所以少搬数据就是直接省能耗。

Innovation

Row-stationary dataflow

Instead of optimizing only one reuse type, Eyeriss tries to balance input reuse, weight reuse, and partial-sum reuse. 创新点在于它不是只偏向一种 stationary，而是更平衡地吃到多种复用收益。

Why It Fits ML/DL

CNNs have structured reuse

Convolution repeatedly reuses nearby activations, filter weights, and partial sums, so a reuse-aware dataflow is a natural match. 卷积天生就有局部窗口和重复使用，因此 Eyeriss 特别适合 CNN。

Best exam angle: Eyeriss is good because it improves energy efficiency by organizing computation around reuse. 它的强点不是“算得更神秘”，而是“用更少的数据移动完成卷积”。

TPU

TPU is an ASIC designed around neural-network matrix operations. Its strength is not general-purpose flexibility, but extremely efficient high-throughput dense linear algebra. TPU 是围绕神经网络矩阵运算定制的专用芯片，核心优势是大规模矩阵乘法吞吐率高。

What It Is

Matrix-centric ML accelerator

The TPU treats matrix multiplication as the main event because many DNN layers reduce to GEMM-like computation. 它把矩阵乘法当作主战场，因为很多神经网络层最终都能归结为矩阵运算。

Why It Is Good

High throughput with regular dataflow

Regular movement through the array enables steady reuse and pipelined computation. 数据按规则流过阵列，适合流水化执行，也方便提高吞吐。

Innovation

Systolic-array-style execution

Data moves rhythmically across many processing elements, so computation happens where data already is. 创新点是让数据在阵列里有规律地流动，边流边算，而不是频繁往返大内存。

Why It Fits ML/DL

DNN workloads are highly regular

Fully connected layers, attention projections, and many convolutions can be organized into matrix multiplies. 神经网络里大量核心算子都规则、重复、并行度高，非常适合 TPU 这种规则阵列。

Best exam angle: TPU is powerful because it maps ML computation onto a highly regular matrix engine, which improves throughput and reuse. 它强在“矩阵计算和硬件结构非常对口”。

GPU Architecture

A GPU is effective for ML because it can run many similar operations on many data elements at the same time. GPU 适合 ML，不是因为“万能”，而是因为 ML 里有大量可并行、重复的计算。

What It Is

Massively parallel throughput processor

A GPU is designed to keep many threads in flight so arithmetic units stay busy. GPU 的设计目标是让大量线程同时工作，尽量喂满计算单元。

Why It Is Good

High parallelism, mature software stack

It supports large-scale parallel kernels and has strong compiler/library ecosystems. 它不光并行度高，而且软件生态成熟，所以深度学习非常依赖 GPU。

Innovation

SIMT execution with programmable hierarchy

Threads are exposed to programmers, but execution is grouped efficiently in warps. 创新点是把线程编程模型和底层成组执行结合起来，兼顾易编程和高吞吐。

Why It Fits ML/DL

ML kernels have huge data parallelism

Vector ops, GEMM, convolution, reduction, and tensor kernels all contain many repeated operations on different data. ML/DL 的很多算子都天然数据并行，所以 GPU 能很好吃满这些工作负载。

Concept Simple explanation Why it matters
ThreadSmallest logical worker 最小逻辑执行单位Lets a kernel map many independent data elements to many workers.

How to read this

How a thread knows its piece of the work

threadIdx.x is the thread's local position inside a block, blockIdx.x is the block's position inside the grid, and blockDim.x is the number of threads per block. 中文直白版就是：线程先知道自己在本组里的编号，再结合这个组在全局里的位置，算出自己该处理哪一段数据。

Best exam angle: GPUs are good for ML because they combine huge data parallelism with memory hierarchies like shared memory that support reuse. GPU 不是只靠“线程多”，而是靠“线程多 + 层次化存储 + 成熟编程模型”。
One clean comparison

GPU vs TPU vs Eyeriss

GPU: flexible, programmable, strong ecosystem. TPU: more specialized for matrix-heavy ML throughput. Eyeriss: more explicitly optimized for CNN data reuse and energy efficiency. 中文抓手：GPU 重灵活，TPU 重矩阵吞吐，Eyeriss 重卷积复用和能效。

2. Different Loss Functions

loss function 是模型训练时的目标函数。模型每做一次预测，都要通过 loss 变成一个“错了多少”的数。训练算法不是直接看 accuracy，而是通过 loss 来更新参数。

为什么不同任务要用不同 loss

slides 里很强调这一点：回归关心数值误差，所以 MSE 很自然；分类关心类别和概率，所以 cross-entropy 更自然；0/1 loss 虽然直观，但不可导，不适合做梯度优化。

MSE / SE

回归常用

平方误差会放大大误差，所以适合连续数值预测。它可导，训练方便，但对分类不自然。

MAE / AE

更直观的平均偏差

对异常值没那么敏感，解释性强，但优化性质不如 MSE 平滑。

0/1 loss

适合评价，不适合训练

对了就是 0，错了就是 1。它最直观，但不可导，而且不区分“错一点”和“错得离谱”。

Hinge / Logistic / Cross-Entropy

分类训练的主力

hinge 强调 margin；logistic 和 cross-entropy 强调概率和高置信度错误的惩罚。

Cross-entropy 为什么在分类里这么重要

如果真实类别是某一类，而模型给这一类的概率很低，那么 cross-entropy 会给出很大损失。它会推动模型不仅“分类对”，还要“对正确类别有高置信度”。这正是 slides 里反复强调的点。

必背：0/1 loss 常用于最终评价；cross-entropy 常用于训练。原因不是“老师规定这样”，而是因为前者不可导、后者更适合梯度优化。

3. Gradient Descent Algorithm and Variations

你可以把 gradient descent 理解成：站在山坡上，想往谷底走。梯度告诉你哪里是“上坡最快方向”，所以你往相反方向迈一步。

θt+1 = θt − η∇L(θt)

每一项都要会解释

  • θ 是参数，也就是模型真正会被训练改变的量。
  • L 是 loss，代表当前错得多不多。
  • ∇L 是梯度，表示 loss 增长最快的方向。
  • η 是 learning rate，决定每一步走多大。

变体之间怎么比较

方法 优点 缺点 考试里一句话
Batch GD 稳定 慢、内存和计算开销大 每次看全部样本再更新
SGD 更新频繁，单次开销小 噪声大、轨迹抖 每次只看一个样本
Mini-batch 兼顾稳定和效率 仍要选 batch size 现实训练最常见
Momentum 减少震荡，加快收敛 多超参数 像球带惯性往下滚
Adam 自适应学习率，常用 不是一切场景都最优 Momentum 加自适应学习率
高频题：如果问 learning rate 太大或太小的后果，答“太大可能震荡或发散，太小会收敛过慢”。这类题很稳。

4. Convolution and CNNs

slides 对 CNN 的讲法很形象：第一层识别边缘，第二层识别角和轮廓，再往后识别部件和整体。你最好把这个层次感直接记住。

卷积操作本质

一个小 filter 在输入上滑动。每到一个位置，就把 filter 覆盖到局部区域上，逐元素相乘再求和，得到输出 feature map 中的一个值。

卷积为什么比全连接更适合图像

Local connectivity

图像的局部像素之间强相关，所以每个输出只看局部区域更合理。

Weight sharing

同一个 filter 在整张图上复用，所以参数大幅减少。

Hierarchy of features

从边缘到轮廓，再到部件和物体，特征是逐层组合出来的。

更自然的归纳偏置

网络结构本身就假设图像有空间结构，因此更适合图像任务。

6. Performance Evaluation and Roofline Analysis

这一块最容易被误解成“只是一个图”。其实它在考你：到底是算力不够，还是搬数据太慢。

从 slides 的角度理解 roofline

老师的 slides 文本里虽然没有大量直接出现 roofline 公式，但一直在讲 cache、bandwidth、tiling、locality、data movement。这些正是 roofline 的底层直觉。

  • Arithmetic intensity 低：每搬一点数据只做很少计算，说明复用差，更可能 memory-bound。
  • Arithmetic intensity 高：每份数据被多次使用，更可能接近 compute-bound。
  • Tiling 的意义：提高 reuse，从而提升 arithmetic intensity。
考试表达模板：Roofline model 帮助我们同时考虑峰值算力和内存带宽。它用 arithmetic intensity 判断一个 workload 更像是 memory-bound 还是 compute-bound，并指导优化方向。

7. Matrix Operations and Transformations: GEMM, Strassen, Winograd

很多神经网络运算最后都想办法转成 GEMM。原因不是数学优雅，而是 GEMM 已经被软件库和硬件做到了极高优化。

GEMM 为什么是中心

  • 矩阵乘法有成熟高性能实现，如 BLAS、cuBLAS、MKL。
  • cache、tiling、SIMD、tensor cores 等优化都和 GEMM 天然匹配。
  • 很多卷积实现最终都绕回矩阵乘法。

Strassen 和 Winograd 的考试重点

你不需要背推导，但要记住：Strassen 是减少矩阵乘法次数，Winograd 是减少小卷积核乘法次数。 同时要知道两者都有代价：额外加法、实现复杂度、数值问题、并非所有场景都赢。

从 slides 提炼：Strassen 更像理论上很强但实现上有门槛；Winograd 更像在 3x3 一类小卷积上很实用，但有变换和精度代价。

8. Kernel Implementation, Toeplitz Matrix, img2col

这里的 kernel 不是卷积核，而是计算内核实现。考试想看你能不能把数学公式和真实实现联系起来。

Toeplitz 的作用

Toeplitz matrix 的意义不是让你现场把大矩阵写出来，而是让你明白：卷积可以被表达成一种特殊的矩阵乘法。这是卷积转 GEMM 的概念桥梁。

img2col 的完整理解

  • 把每个滑动窗口抽出来。
  • 每个窗口拉平成为一列。
  • 所有列拼成大矩阵。
  • 把卷积核也重排成矩阵。
  • 最后变成 GEMM。
优点

复用成熟矩阵乘法库

这让卷积实现可以直接享受 GEMM 优化带来的性能红利。

缺点

中间矩阵膨胀

很多重叠区域被重复展开，增加内存容量和带宽压力。

9. Dataflow and Stationariness

这部分是后面 TPU、Eyeriss、甚至 GPU shared memory 优化的共同语言。你要学会的问题不是“名字是什么”，而是“到底固定什么最划算”。

统一原则

stationary 的意思是：尽量让某类数据留在近处不动，让它被多次使用，减少昂贵的数据移动。

三种主流 stationary 的考试说法

  • Weight-stationary：权重尽量留在 PE 或近处 buffer，让输入流过来。适合权重复用高的情况。
  • Output-stationary：partial sums 留在本地累加，减少中间结果写回。
  • Input-stationary：输入激活留在本地，让多个权重或多个输出重用同一输入。
别死记名字：真正的核心不是定义，而是它们都在试图减少带宽压力和数据搬运能耗。

10. TPU

TPU 是 Google 为神经网络矩阵计算设计的专用 ASIC。slides 里很强调它要解决的问题：DNN 计算太大，传统 CPU/GPU 不够专门，数据中心需要更高吞吐和更低能耗。

你必须会的 TPU 关键词

Systolic array

数据和权重按规则流经 PE 阵列，边流边算，局部复用很强。

Matrix multiply unit

TPU 的核心计算单元，专门为大规模矩阵乘法服务。

Weight stationary 倾向

slides 里明确提到权重复用和 weight FIFO，这正是 TPU 的关键思路之一。

Vector unit

负责 ReLU、softmax、normalization 等逐元素或向量操作。

考试模板：TPU 的核心不是通用并行，而是围绕矩阵乘法设计的专用高吞吐数据流系统，代表结构是 systolic array。

11. Basic Architecture of Eyeriss

从 slides 抽出来，Eyeriss 最重要的一句话是：AI 里搬数据的能耗比 MAC 高很多很多。 所以 Eyeriss 不只是追求快，而是追求能效。

你一定要会写出来的词

  • Row-stationary dataflow
  • PE array
  • local scratchpad
  • NoC
  • global buffer
  • off-chip DRAM

Row-stationary 为什么强

它不是只固定权重，也不是只固定输出，而是尽量在行级别上同时兼顾 input reuse、weight reuse 和 partial sum reuse。也正因为这种“更平衡”的复用方式，Eyeriss 在 CNN 上有很强的能效表现。

Eyeriss 和 TPU 不要混：TPU 更像“矩阵吞吐怪兽”；Eyeriss 更像“CNN 数据流和能效专家”。

12. GPU Architecture, Warp, Block, Thread ID

GPU 这部分 slides 的主线是：编程上你写的是 threads，底层硬件执行更像 SIMT。很多执行同一条指令的 threads 会被硬件动态组织成 warp。

最该记住的层级关系

→
Warp

Block

Grid

你要会解释的三个点

  • Thread：最小逻辑执行单位，每个线程通常处理不同数据。
  • Warp：一组同时执行同一条指令的 threads，是硬件执行的重要单位。
  • Block：一组 threads，可以共享 shared memory，并在 block 内协作。

Thread ID 公式怎么读

global_id = blockIdx.x × blockDim.x + threadIdx.x

公式不是拿来吓人的。它只是告诉你：一个线程要先知道自己在 block 里的位置，再知道 block 在 grid 里的位置，然后合成全局编号，决定自己处理哪一部分数据。

Shared memory 为什么重要

因为它比 global memory 快得多，block 内线程可以把重复使用的数据先搬进 shared memory，再多次复用，从而减少慢速全局访存。这和前面讲的 tiling、dataflow、reuse 是一条逻辑线。

考试不要只写定义：如果问 warp 或 shared memory，最好补一句“它们的存在是为了提高并行执行效率和数据复用效率”。

Detailed Slide-Based Notes

This section is intentionally much denser. It is written to feel closer to actual lecture-note depth, not just a study summary. Use it when the short cards and tables are not enough.

1. Machine Learning Fundamentals: what the model is really doing

At the most basic level, machine learning is about learning a function that maps an input to an output. In the early lectures, the important point is not just the equation itself, but the training loop around it: data goes in, the model makes a prediction, that prediction is compared to the true answer using a loss function, and then the parameters are updated. If you remember only one thing from the fundamentals, remember that the model is not “finding truth directly”; it is repeatedly minimizing a training objective.

For classification, the output is a discrete class label. The model is trying to separate inputs into categories such as spam vs not spam, or cat vs dog. For clustering, there are no labels. The algorithm tries to discover structure in the data by grouping similar points together. This is why classification is supervised learning and clustering is unsupervised learning.

The slides treat this distinction as foundational because many later choices depend on it. The choice of loss, the choice of evaluation metric, and even the interpretation of model output all depend on whether the task is classification or clustering.

Slide-aligned exam insight: if a question asks for “basic models,” do not overcomplicate the answer. A linear model, logistic regression, and k-means are enough to anchor your explanation unless the question explicitly asks for more.

Basic models you should be able to explain

Model Main idea Why it matters for the exam
Linear model The output is a weighted sum of the input plus a bias. It is the simplest trainable model and the cleanest starting point for talking about parameters and gradients.
Logistic regression A linear score is turned into a probability for binary classification. It connects classification, probability, and logistic loss.
K-means Assign points to the nearest center, then update the centers repeatedly. It is the canonical clustering example and makes unsupervised learning concrete.

2. Loss Functions: why training needs more than “correct or wrong”

A loss function converts prediction quality into a number that optimization can act on. This sounds obvious, but it is one of the most important ideas in the course. The slides emphasize that the training objective is not necessarily identical to the final metric. For example, 0/1 loss is intuitive for evaluation because it directly measures correctness, but it is not smooth and not differentiable in a useful way for gradient-based training.

This is why classification often uses logistic loss or cross-entropy instead of 0/1 loss. Cross-entropy does not just ask whether the answer is correct; it also asks whether the model assigned high probability to the correct class. If the correct class receives very low probability, the loss becomes very large. That gives the optimizer a strong signal.

For regression, MSE is natural because the target is a numerical value and squared deviation is meaningful. For classification, MSE is often a poor match because classification is about discrete decisions and probability distributions rather than raw numeric distance.

How to talk about major losses

  • MSE: appropriate for regression, differentiable, punishes large numerical errors heavily.
  • 0/1 loss: intuitive for final correctness, but too coarse and not suitable for direct gradient optimization.
  • Hinge loss: useful when the model should not only classify correctly but also keep a margin from the decision boundary.
  • Logistic loss: a smooth classification-friendly objective, especially natural for binary settings.
  • Cross-entropy: the standard answer for multi-class probabilistic classification.
What instructors like to ask: “Why not use MSE for classification?” The robust answer is: because classification is about correct categories and confidence/probabilities, not about minimizing raw numeric distance between labels.

3. Gradient Descent and Its Variations

The slides treat gradient descent as the general training engine. The intuition should be completely clear: if the gradient points toward the steepest increase in loss, then moving in the negative gradient direction decreases the loss. The learning rate controls how large that step is.

The biggest conceptual risk for beginners is treating the update equation like a memorization item instead of an algorithm. It is an algorithm. Each training step means: compute prediction, compute loss, compute gradient, update parameters. This cycle repeats until the parameters settle into a region with lower loss.

The variations matter because real training has practical constraints. Batch gradient descent is stable but expensive. SGD is fast and noisy. Mini-batch is the practical compromise and is what you should assume unless a problem explicitly says otherwise. Momentum keeps part of the previous direction to reduce oscillation. Adam is widely used because it combines momentum-like behavior with adaptive step sizes.

Common exam angles

Question type Good answer direction
Why is there a negative sign? Because we want to move opposite the direction of greatest loss increase.
What happens if the learning rate is too large? The updates may overshoot, oscillate, or diverge.
Why use mini-batches? They balance stability, efficiency, and hardware-friendly parallelism.
Why use momentum? To smooth updates and reduce zig-zagging in narrow valleys.

4. CNNs and Convolution: the representation story

The CNN lectures are not just teaching the convolution operation mechanically. They are building a story of representation. Early layers learn local edges and simple patterns. Middle layers combine those into contours and parts. Later layers combine parts into object-level recognition. That is why CNNs are not just “matrix operations on images”; they are hierarchical feature extractors.

Convolution is attractive because it builds in assumptions that are appropriate for images. Images have local structure, so local connectivity makes sense. The same useful pattern can appear in many places, so weight sharing makes sense. Together these reduce parameter count and make learning more data-efficient than a fully connected alternative.

Padding, stride, and pooling are all part of how information is shaped as it flows through the network. Padding controls boundary behavior and output size. Stride controls how aggressively the feature map shrinks. Pooling reduces spatial dimensions and helps with translation tolerance, though it also discards detail.

What not to miss in the CNN slides

  • In deep learning, “convolution” is often implemented more like cross-correlation, but the name convolution is still used.
  • Multi-channel convolution means each filter spans all input channels and the contributions are summed.
  • 3D convolution is not the same thing as 2D convolution on a 3-channel RGB image.
  • Global pooling is important because it can replace large fully connected heads and reduce parameter count.

5. Famous CNN and DNN Models: why each one matters historically

The right way to study the famous models is as a progression of ideas. LeNet proves CNNs are useful. AlexNet proves deeper CNNs plus GPUs can dominate large-scale image recognition. VGG shows that repeating small filters in a very regular architecture can be very effective. GoogLeNet shows that multi-branch multi-scale processing can improve representational efficiency. ResNet solves the major depth-related training difficulty by providing shortcut paths for gradients. MobileNet pushes efficiency and deployability.

The slides on AlexNet are especially important because they link model design to hardware reality. Group convolution in AlexNet was not only a modeling choice; it was also shaped by the practical limitation of splitting work across two GPUs. That is a very EE508-style point: architecture and implementation are connected.

ResNet is another key lecture highlight. It matters not just because it is famous, but because it gives a concrete answer to why deeper plain networks can perform worse: optimization difficulty and degradation. The skip connection provides a cleaner path for gradient flow.

6. Performance Evaluation and Roofline: what the slides are really emphasizing

Even when the term “roofline” is not shown everywhere, the performance lectures are preparing you for roofline-style reasoning. The recurring themes are cache, locality, bandwidth, and data movement. That means the real question is not only “how many operations are there?” but also “how often do we have to fetch data?”

Arithmetic intensity gives a compact way to talk about this: how much computation is performed per unit of data movement. If arithmetic intensity is low, performance is more likely to be memory-bound. If arithmetic intensity is high, the hardware has a better chance of approaching compute limits.

This is why tiling shows up repeatedly. Tiling is not just a coding trick; it is a way to increase locality and reuse, which effectively improves arithmetic intensity from the machine’s point of view.

Strong exam phrasing: “Roofline analysis helps distinguish whether a workload is limited more by compute throughput or memory bandwidth, and arithmetic intensity is the key bridge between the two.”

7. Matrix Operations and Transformations: why this block exists in the course

This block is teaching you that deep learning computation is often reorganized into forms that hardware and libraries handle well. GEMM is central because it is the most heavily optimized dense computation primitive in modern numerical computing. If a convolution can be lowered into GEMM, it can inherit an enormous ecosystem of optimizations.

Strassen matters because it is the classic example showing that even matrix multiplication itself is not fixed. The usual cubic formulation is not the only possibility. By reducing multiplication count at the cost of additional additions and structural complexity, Strassen improves asymptotic complexity. The slides also emphasize why that does not automatically make it dominant in practice: memory overhead, precision concerns, and large constant factors matter.

Winograd plays a similar conceptual role for convolution. It shows that for small kernels, especially 3×3, one can reduce multiplication count through a transform-based approach. But again, there is no free lunch. There is extra transform work, there may be numerical sensitivity, and its benefits depend strongly on kernel size and layer shape.

8. Kernel Implementation, Toeplitz, and img2col

This is where the course moves from “what operation is being computed” to “how the operation is actually implemented.” Toeplitz is important conceptually because it shows that convolution can be expressed as a structured matrix multiplication. It is not important because you will literally store giant Toeplitz matrices in practice. In fact, one of the lessons is that explicit structured expansion can be wasteful.

img2col is the practical bridge. It extracts each receptive field, flattens it into a column, and turns the convolution into a matrix multiplication problem. This is powerful because it lets implementation reuse optimized GEMM kernels. But it is also expensive because overlapping windows create duplicated data in the intermediate representation.

That trade-off is exactly the kind of systems thinking this course wants you to develop: a mathematically elegant reformulation can still be costly in memory footprint and bandwidth. So the “best” implementation is not determined by arithmetic count alone.

9. Dataflow and Stationary Schemes

Dataflow is the language used to describe how inputs, weights, and partial sums move through an accelerator. Stationary schemes are specialized versions of this idea in which one type of data is kept close to the compute units as much as possible. The goal is always the same: reduce expensive movement and maximize reuse.

Weight-stationary keeps weights near the processing elements and streams inputs through. Output-stationary keeps partial sums local so that accumulation does not require repeated writes and reads. Input-stationary keeps activations close so multiple filters or outputs can reuse them. None of these is “the best” in all situations. Each one reflects a different reuse priority.

The slides also connect stationary schemes to reuse distance and bandwidth implications. That is a very important point: dataflow is not only a conceptual mapping choice; it directly affects energy and performance.

10. TPU: what makes it different from generic parallel hardware

TPU is not simply “a faster processor.” The reason it appears in this course is that it is designed around a dominant ML workload pattern: large-scale matrix multiplication. The systolic array is the architectural center because it supports regular, pipelined, highly local data reuse. Inputs and weights enter in structured ways, multiply-accumulate work is done as values move through the array, and the architecture is shaped around throughput and efficiency.

The TPU slides also emphasize that the TPU is part of a larger system. There are host interactions, memory hierarchy decisions, vector units for non-matrix operations, and workload mapping questions. That means a good answer about TPU should mention not just “systolic array,” but also why such an array is well-matched to DNN matrix workloads and how it reduces the pressure on external memory access.

11. Eyeriss: why row-stationary is a big deal

Eyeriss is central because it turns the abstract idea of dataflow into a concrete accelerator architecture. The headline insight is that moving data to and from off-chip DRAM is vastly more expensive than the MAC operations themselves. Therefore, the architecture should be built around reducing movement, not simply increasing raw arithmetic throughput.

Row-stationary is important because it is more balanced than a pure weight-stationary or pure output-stationary strategy. Instead of fully optimizing only one reuse type, it tries to capture multiple forms of reuse at once: weights, activations, and partial sums. That is why the Eyeriss lecture is not just about naming blocks such as PE arrays and global buffers; it is about understanding why that combination supports efficient CNN execution.

12. GPU Architecture: thread, warp, block, and why SIMT matters

The GPU lecture is trying to bridge programming and hardware execution. From the programmer’s perspective, you write many threads that run the same code on different data. From the hardware perspective, these are dynamically organized into warps. That is why GPUs are often described as SIMT machines: they expose a thread-based programming model but execute in a grouped, SIMD-like way under the hood.

You should know the hierarchy clearly: threads are the smallest logical units, warps are execution groups of threads following the same instruction path, blocks are programmer-visible groups of threads that can cooperate and share on-chip memory, and grids are collections of blocks for a kernel launch. The thread ID formula tells each thread which portion of the work it is responsible for.

Shared memory is important because it is the GPU equivalent of “keep data near compute and reuse it.” That directly links the GPU section back to the earlier performance, tiling, and dataflow themes. A good answer about shared memory should mention both speed and reuse.

13. What the instructor is not asking you to do

The exam statement explicitly says you do not need to memorize long formulas like Winograd. That should change how you study. You are not preparing for a symbolic derivation exam. You are preparing for a concept, systems, and explanation exam. You should be able to describe the purpose of Winograd, not derive every transform matrix from memory. You should be able to explain the role of row-stationary dataflow, not redraw an entire chip floorplan perfectly from memory.

In other words, depth matters, but the right kind of depth. The right kind of depth is understanding the motivation, mechanism, and trade-off of each concept.

14. Best answer style for this specific midterm

Because the exam is in-person, closed-book, and paper-based, you should practice speaking and writing compact technical explanations. For most questions, the strongest structure is:

  • Define the concept clearly.
  • State why it is used.
  • State one or two major trade-offs or comparisons.

That structure works for loss functions, optimization methods, CNN models, dataflow schemes, and hardware architectures.
