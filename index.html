1. Gradient Descent — 梯度下降

想象你蒙着眼站在山上，想走到最低点。你能做的就是摸一下脚下的坡度（梯度），然后往下坡方向走一小步。重复这个过程，最终到达谷底（Loss 最小）。

公式只有一个

$$w_{new} = w_{old} - \alpha \cdot \frac{\partial L}{\partial w}$$

拆开来理解：

  • $w$ = 你要调的参数（weight）
  • $\alpha$ = 步长（Learning Rate），走多远。你自己设的，不是算出来的
  • $\frac{\partial L}{\partial w}$ = Loss 对 w 的偏导 = 坡度方向。这个要算
  • 减号 = 往反方向走（坡度指向上坡，我们要下坡）

手把手：怎么算梯度？

假设用 MSE Loss，模型是线性的 $\hat{y}=w_1 x_1 + w_2 x_2$：

$$L = (\hat{y} - y)^2$$

对 $w_1$ 求偏导（Chain Rule）：

$$\frac{\partial L}{\partial w_1} = 2(\hat{y}-y) \cdot \frac{\partial \hat{y}}{\partial w_1} = 2(\hat{y}-y) \cdot x_1$$

规律：MSE 梯度 = 2 × 误差 × 对应的输入

例题：一步 GD 更新

已知 $\hat{y}=3x_1+2x_2$, 数据点 $(x_1,x_2,y)=(1,2,10)$, $\alpha=0.1$, 用 MSE。求更新后 $w_1$。

Step 1: 算预测值
$\hat{y}=3(1)+2(2)=3+4=7$
Step 2: 算误差
$\hat{y}-y=7-10=-3$（预测小了3）
Step 3: 算梯度
$\frac{\partial L}{\partial w_1}=2(-3)(1)=-6$
（注意：$x_1=1$，乘对应的输入）
Step 4: 更新
$w_1=3-0.1\times(-6)=3+0.6=\mathbf{3.6}$
（梯度为负 → 减去负数 → w 变大 → 预测值变大 → 更接近 $y=10$）

常见错误

  • 忘记乘 $x$：梯度公式里的 $x_1$ 很多人漏掉
  • 符号搞反：是 $w - \alpha \cdot gradient$，不是加号
  • 不知道对哪个 $w$ 求导：题目问 $w_1$ 就乘 $x_1$，问 $w_2$ 就乘 $x_2$

三种 GD — 考概念

方法每步用多少数据特点
Batch GD全部 N 个样本梯度准确，但一步很慢
SGD随机 1 个样本一步很快，但方向不稳（噪声大）
Mini-batch一批（如 32 个）折中，实际最常用

Learning Rate 影响

$\alpha$ 太大

步子太大 → 跨过最低点 → 来回震荡 → 可能发散（Loss 越来越大）

$\alpha$ 太小

步子太小 → 收敛极慢 → 可能卡在局部最小

2. Loss Functions — 怎么衡量"错多少"

Loss = 预测和真实值之间的差距。不同任务用不同的 loss，但核心思想一样：预测越准，loss 越小。

回归任务（输出连续值）

MSE — 均方误差

$$L=\frac{1}{N}\sum_{i=1}^{N}(\hat{y}_i-y_i)^2$$

平方的意义：大误差被放大惩罚（差 2 比差 1 的惩罚大 4 倍）

求导简单：$\frac{\partial}{\partial \hat{y}}=2(\hat{y}-y)$

MAE — 均绝对误差

$$L=\frac{1}{N}\sum_{i=1}^{N}|\hat{y}_i-y_i|$$

对 outlier 更不敏感（差 2 只比差 1 惩罚 2 倍，不是 4 倍）

分类任务（输出概率）

Cross-Entropy — 交叉熵（多分类）

$$L=-\sum_{c=1}^{C} y_c \log \hat{y}_c$$

$y_c$ 是 one-hot（只有正确类=1，其余=0），所以实际只算：$L=-\log\hat{y}_{correct}$

如果预测正确类概率=0.9，$L=-\log(0.9)=0.105$（很小✓）

如果预测正确类概率=0.1，$L=-\log(0.1)=2.303$（很大✗）

Binary Cross-Entropy（二分类）

$$L=-[y\log\hat{y}+(1-y)\log(1-\hat{y})]$$

$y=1$ 时只剩 $-\log\hat{y}$；$y=0$ 时只剩 $-\log(1-\hat{y})$

3. Chain Rule & Backprop — 怎么算深层网络的梯度

一个多层网络就像一条流水线：输入 → 第一层 → 第二层 → ... → Loss。
Chain Rule 说：总变化 = 每一段变化的乘积。
就像：汇率换算 1美元→7人民币→0.7港币，总换算率 = 7 × 0.1 = 0.7。

Chain Rule 公式

如果 $L$ 依赖 $\hat{y}$，$\hat{y}$ 依赖 $z$，$z$ 依赖 $w$：

$$\frac{\partial L}{\partial w}=\frac{\partial L}{\partial \hat{y}}\cdot\frac{\partial \hat{y}}{\partial z}\cdot\frac{\partial z}{\partial w}$$

就是把每一段的导数乘起来。

手算例题：2 层网络

网络：$z=w \cdot x$，$\hat{y}=\text{ReLU}(z)=\max(0,z)$，$L=(\hat{y}-y)^2$
已知 $w=2, x=3, y=5$。求 $\frac{\partial L}{\partial w}$。

Step 1: Forward（正向算一遍）
$z=2 \times 3=6$
$\hat{y}=\text{ReLU}(6)=6$（因为 6>0）
$L=(6-5)^2=1$
Step 2: 从后往前，逐段求导
$\frac{\partial L}{\partial \hat{y}}=2(\hat{y}-y)=2(6-5)=2$
Step 3: ReLU 那段
$\frac{\partial \hat{y}}{\partial z}=1$（因为 $z=6>0$，ReLU 导数=1）
（如果 $z<0$，ReLU 导数=0，梯度就断了！→ 这就是 dead ReLU 问题）
Step 4: 线性那段
$\frac{\partial z}{\partial w}=x=3$
Step 5: 乘起来
$\frac{\partial L}{\partial w}=2 \times 1 \times 3=\mathbf{6}$

Vanishing Gradient 问题

Sigmoid 导数最大=0.25。10 层网络：$0.25^{10}\approx 0.000001$，梯度几乎为 0！

解决方法
  • ReLU：正区间导数=1，不会衰减
  • BatchNorm：归一化每层输出 $\hat{x}=\frac{x-\mu}{\sqrt{\sigma^2+\epsilon}}$
  • 好的初始化：Xavier（tanh），He（ReLU）

4. CNN Output Size — 考试必出的计算题

一个 filter 在输入上滑动，每滑一次产生一个输出。输出大小 = 能滑多少次 + 1。

万能公式

$$\text{Out}=\left\lfloor\frac{\text{In} - F + 2P}{S}\right\rfloor + 1$$

每个字母的意思：

符号英文中文典型值
$\text{In}$Input size输入的宽或高28, 32, 224...
$F$Filter size卷积核大小3, 5, 7, 11...
$P$Padding输入外面加几圈 00, 1, 2...
$S$Stride每次滑几格1, 2

做题套路（3步）

例 1：基本题

In=7, F=3, P=0, S=1

Step 1: 代入分子 In - F + 2P = 7 - 3 + 0 = 4
Step 2: 除以 S 4 / 1 = 4
Step 3: 加 1 4 + 1 = 5

例 2：有 Padding 和 Stride

In=32, F=5, P=2, S=2

Step 1: 32 - 5 + 2×2 = 32 - 5 + 4 = 31
Step 2: 31 / 2 = 15.5 → 下取整 → 15
Step 3: 15 + 1 = 16

例 3：Quiz 原题 — 什么时候 Invalid？

In=3, F=11, P=0, S=1

Step 1: 3 - 11 + 0 = -8（负数！）
结论: Filter 比输入还大，放不下 → Invalid！

Same Padding — 让输出=输入

当 $S=1$ 时，想让 Out = In，需要：

$$P=\frac{F-1}{2}$$

例：In=28, F=7, S=1，求 Same Padding

$P = (7-1)/2 = 3$
验证：$\lfloor(28-7+6)/1\rfloor+1 = 27+1 = 28$ ✓ 输出=输入

多通道 — 深度怎么算？

记住两点
  1. 一个 filter 的深度 = 输入通道数 $C$（必须匹配）
  2. 用 $M$ 个 filter → 输出深度 = $M$
输入 $H\times W\times C$，$M$ 个 $R\times S\times C$ filter → 输出 $H'\times W'\times M$
$H'$ 和 $W'$ 用上面的公式算

Quiz Q4 原题：2D 多通道

Input $3\times3\times3$（3通道），3 个 $2\times2\times3$ filter，S=1, P=0

宽/高: $\lfloor(3-2+0)/1\rfloor+1=2$
深度: 3 个 filter → 深度 = 3
输出: $2\times2\times3$

Output Size 常见错误

  • 忘记 2P：Padding 是两边都加，所以是 $2P$ 不是 $P$
  • 忘记下取整：结果有小数时必须 floor
  • 通道数搞错：filter 深度=输入通道（不影响空间大小），输出深度=filter个数

5. Convolution vs Cross-Correlation

数学上的卷积 = 先把 kernel 翻转 180°，再滑动相乘求和。
CNN 的"卷积" = 不翻转，直接滑动相乘求和（其实是 cross-correlation）。
为什么没关系？因为 kernel 是学出来的，翻不翻转学到的值不同而已，效果一样。

数值例子

Input: [1, 2, 3, 4, 5], Kernel: [a, b, c]

Cross-Correlation（CNN 用的）
output[0] = 1×a + 2×b + 3×c
output[1] = 2×a + 3×b + 4×c
kernel 不动，按原顺序 [a, b, c] 滑
Convolution（数学定义）
先翻转 kernel → [c, b, a]
output[0] = 1×c + 2×b + 3×a
output[1] = 2×c + 3×b + 4×a
考试答题模板

Convolution 需要 flip（翻转）kernel，有交换律。CNN 实际使用 cross-correlation（不翻转），因为 kernel 是通过学习得到的，翻转与否不影响模型表达能力。

6. im2col — 把卷积变成矩阵乘法

卷积本质是"滑动窗口 + 求和"。im2col 把每个窗口拉成一列，所有窗口拼成一个大矩阵，然后用一次矩阵乘法算出所有输出。GPU 做矩阵乘特别快（有 BLAS 库优化），所以这样反而更快。

具体例子

Input $4\times4$，Filter $2\times2$，S=1, P=0 → Output $3\times3$（9个位置）

Step 1: Filter 拉成行向量
Filter $\begin{bmatrix}a&b\\c&d\end{bmatrix}$ → 行向量 $[a, b, c, d]$（1×4）
Step 2: 每个滑动窗口拉成列
窗口1（左上2×2）→ 列1
窗口2（右移一格）→ 列2
...共 9 个窗口 → 拼成 4×9 矩阵
Step 3: 矩阵乘
$[1\times4] \times [4\times9] = [1\times9]$ → reshape 成 $3\times3$ 输出

im2col 的代价

优点

卷积→GEMM，利用高度优化的矩阵乘库，计算速度快

缺点

窗口重叠 → 同一数据被复制多次 → 内存占用↑
FLOPs 不变，但 memory traffic↑ → AI 下降

7. Roofline Model — 性能上限分析

一个程序的速度要么被计算能力卡住（CPU/GPU 算不过来），要么被内存带宽卡住（数据搬不过来）。Roofline 模型就是判断你被哪个卡住了。

核心概念：Arithmetic Intensity (AI)

$$AI = \frac{\text{计算量 (FLOPs)}}{\text{数据传输量 (Bytes)}} \quad\text{单位: FLOP/Byte}$$

AI 衡量的是：每搬 1 字节数据，能做多少次运算。

  • AI 高 → 搬一次数据反复用 → compute-bound
  • AI 低 → 搬很多数据只算一点 → memory-bound

Roofline 公式

$$\text{性能} = \min(\text{Peak GFLOP/s}, \quad AI \times \text{Peak BW})$$

性能取两者中较小的（被哪个卡住就是哪个）。

判断方法：算 Transition Point

$$AI_{trans} = \frac{\text{Peak GFLOP/s}}{\text{Peak GB/s}}$$
  • $AI < AI_{trans}$ → Memory Bound（搬数据太慢，算力用不满）
  • $AI \geq AI_{trans}$ → Compute Bound（算力满了，搬数据够快）

例题：判断 bound 并算性能

硬件：Peak = 100 GFLOP/s, BW = 50 GB/s。程序 AI = 1.5

Step 1: 算 Transition
$AI_{trans} = 100/50 = 2$
Step 2: 比较
$AI=1.5 < 2$ → Memory Bound!
Step 3: 算性能
Performance = $AI \times BW = 1.5 \times 50 = \mathbf{75}$ GFLOP/s
（没达到 100 的峰值，被带宽拖后腿）

怎么算 AI — 考试出题套路

FP32 = 4 bytes/element

操作FLOPs数据量AI怎么算的
Vector Add
$C_i = A_i + B_i$		N3N×4=12N B$\frac{N}{12N}=\frac{1}{12}$读 A,B 写 C 各 N 个 FP32
Dot Product
$c=\sum A_i B_i$		2N2N×4+4=8N+4$\approx\frac{1}{4}$N 乘 + N-1 加 ≈ 2N; 读 A,B 写 1 标量
GEMV
$y=Ax$		$2N^2$$(N^2+2N)\times4$$\approx\frac{1}{2}$矩阵 $N^2$ + 向量 $N$ + 输出 $N$
GEMM
$C=AB$		$2N^3$$3N^2\times4$$\frac{N}{6}$$2N^3$ FLOPs, 3 个 $N\times N$ 矩阵
关键洞察

GEMM 的 AI 随 N 增长！ N=600 时 AI=100，远超 transition。所以大矩阵乘法是 compute-bound，GPU/TPU 专门优化它。
而 Vec Add 的 AI 恒=1/12，永远 memory-bound。

例题：GEMM AI 计算

两个 64×64 的 FP32 矩阵相乘

FLOPs: $2 \times 64^3 = 2 \times 262144 = 524288$
Bytes: 3 个 64×64 矩阵 × 4 bytes = $3 \times 4096 \times 4 = 49152$
AI: $524288 / 49152 \approx \mathbf{10.67} \approx 64/6$ ✓

Quiz 的 Systolic Array AI

3×3 Array, 每列 N 个权重

Activation AI:
FLOPs: 每 act 经过 3 PE，每 PE 做 3 MAC → 18 ops。共 N 个 → 18N
Bytes: 3 列各 N 个 FP32 读入 = 3N×4 = 12N
$AI_{act} = 18N/12N = \mathbf{1.5}$
Weight AI:
每个权重被 N 个 act 复用 → FLOPs = 2N
Bytes: 1 个 FP32 = 4
$AI_{wt} = 2N/4 = \mathbf{0.5N}$
N 越大 → 权重复用越多 → AI 越高!

9. SIMD 两种实现方式

SIMD = 一条指令处理多个数据。但具体怎么"多个"？有两种截然不同的方式。
Array Processor（铺开在空间）

多个 PE 同时干活

  • N 个 PE 排成一排
  • 每个 PE 有自己的 ALU + Register
  • 同一时刻，所有 PE 执行相同操作但处理不同数据
  • 一个 cycle 就能处理 N 个数据

关键词：Same time, different space（空间并行）

Vector Processor（排开在时间）

一个 PE 用深流水线连续处理

  • 1 个功能单元做成深度流水线
  • 每 cycle 送入一个新数据
  • 多个数据同时处于流水线不同阶段
  • 启动延迟后，steady state = 1 result/cycle

关键词：Same space, different time（时间并行）

一句话区分

Array = 空间上铺开多个单元，同一时刻并行处理
Vector = 时间上流水一个单元，不同阶段重叠处理

10. Array vs Vector — 详细对比

时序图理解（关键！）

任务：4 个元素，每个要 LD→ADD→MUL→ST

Array Processor（4 个 PE）

T1: LD0 LD1 LD2 LD3 ← 4个PE同时Load T2: AD0 AD1 AD2 AD3 ← 4个PE同时Add T3: MU0 MU1 MU2 MU3 ← 4个PE同时Mul T4: ST0 ST1 ST2 ST3 ← 4个PE同时Store 总共: 4 cycles

Vector Processor（流水线）

T1: LD0 T2: LD1 AD0 T3: LD2 AD1 MU0 T4: LD3 AD2 MU1 ST0 T5: AD3 MU2 ST1 T6: MU3 ST2 T7: ST3 总共: 7 cycles（但只需1个PE!）
对比项ArrayVector
并行方式空间：多 PE 同时时间：流水线重叠
硬件成本高（N 个 PE）低（1 个 PE + pipeline）
延迟低（O(stages)）高开始 → startup + N-1
吞吐N/cycle1/cycle（steady state）
典型GPU SMCRAY-1

11. Memory Banks — 考试重点！

问题：内存读一次要 11 个 cycle（latency=11）。如果连续读，每次都要等 11 cycle，太慢了！
解决：把内存分成 16 个 bank，轮流读。Bank 0 还在忙的时候，去读 Bank 1，Bank 1 忙了读 Bank 2...等读完一圈回来，Bank 0 已经好了。像旋转门一样！

关键条件

要实现每 cycle 读出 1 个元素，需要满足：
  1. Banks ≥ Latency：bank 数量不少于延迟周期数（否则转一圈回来 bank 还没好）
  2. 连续元素在不同 bank：stride=1 时，元素 0 在 bank 0，元素 1 在 bank 1...（交错存储）

Bank 地址计算

$$\text{Bank Number} = \text{Address} \mod \text{NumBanks}$$

例：16 banks, stride=1

元素 0 → bank 0%16 = 0
元素 1 → bank 1%16 = 1
元素 2 → bank 2%16 = 2
...每个元素在不同 bank → 不冲突！ → 每 cycle 读 1 个

例：8 banks, stride=4

元素 0 → bank 0%8 = 0
元素 4 → bank 4%8 = 4
元素 8 → bank 8%8 = 0 ← 和元素 0 同一个 bank!
元素 12 → bank 12%8 = 4 ← 和元素 4 同一个 bank!
结果：只用到 bank 0 和 4 → Bank Conflict！性能骤降

Bank Conflict 发生条件

当 stride 和 bank 数量不互素（有公因数）时，会有冲突：

  • Stride=2, 8 banks → GCD(2,8)=2 → 只用 8/2=4 个 bank → 冲突
  • Stride=3, 8 banks → GCD(3,8)=1 → 用满 8 个 bank → 不冲突
  • Stride=1 → GCD(1,N)=1 → 永远不冲突

CRAY-1 参数（背下来）

参数值为什么
Banks1616 > 11(latency) → 不 stall
Vector Regs8 个 × 64 元素 × 64 bit能存够多向量
Bank Latency11 cyclesSRAM 访问延迟

12. Scalar vs Vector Code — 怎么算 Cycle 数

任务：for(i=0; i<50; i++) C[i] = (A[i]+B[i]) / 2

Scalar 代码

MOVI R0 = 50 ; 1 cycle ← 循环计数器 MOVA R1 = A ; 1 cycle ← A 的起始地址 MOVA R2 = B ; 1 cycle ← B 的起始地址 MOVA R3 = C ; 1 cycle ← C 的起始地址 X: LD R4 = [R1++] ; 11 cycles ← 读 A[i] LD R5 = [R2++] ; 11 cycles ← 读 B[i] ADD R6 = R4 + R5 ; 4 cycles ← 加法 SHR R7 = R6 >> 1 ; 1 cycle ← 右移=除以2 ST [R3++] = R7 ; 11 cycles ← 写 C[i] DECBNZ R0, X ; 2 cycles ← R0--, 不为0则跳回X

Scalar 算周期

初始化: 4 条指令 × 1 cycle = 4 cycles
1 bank 时：每次循环 = 11+11+4+1+11+2 = 40 cycles
（两个 LD 和 ST 不能并行，因为只有 1 个 bank）
总计 (1 bank): 4 + 50 × 40 = 2004 cycles
2 banks 时：A 和 B 在不同 bank → 两个 LD 可重叠！
每次循环 = 11 + 4 + 1 + 11 + 2 + 1(overhead) = 30 cycles
总计: 4 + 50 × 30 = 1504 cycles

Vector 代码

MOVI VLEN = 50 ; 1 cycle ← 向量长度 MOVI VSTR = 1 ; 1 cycle ← stride = 1 VLD V0 = A ; 11+49 = 60 ← 读整个向量 A[0..49] VLD V1 = B ; 11+49 = 60 ← 读整个向量 B[0..49] VADD V2 = V0 + V1 ; 4+49 = 53 ← 向量加 VSHR V3 = V2 >> 1 ; 1+49 = 50 ← 向量右移 VST C = V3 ; 11+49 = 60 ← 写回整个向量

Vector 怎么算周期

向量指令耗时公式: startup + (VLEN - 1)
startup = 该操作的 latency（LD=11, ADD=4, SHR=1, ST=11）
VLEN-1 = 后续元素在流水线中：每 cycle 出 1 个
VLD: 11 + 49 = 60 cycles（第一个等 11 cycle，后面 49 个每 cycle 一个）
VADD: 4 + 49 = 53 cycles
总计: 2 + 60 + 60 + 53 + 50 + 60 = 285 cycles
Speedup: 2004 / 285 ≈ 7× 快!
为什么 16 banks 不 stall？

VLD 每 cycle 读 1 个元素。Stride=1 → 连续访问不同 bank。
Bank latency=11, 有 16 banks (16>11)。
读完 bank 0 → 1 → 2 → ... → 15 → 回到 0 时，bank 0 已经空闲（过了 16>11 cycle）。不 stall！

算周期的常见错误

  • Vector 耗时不是 startup × VLEN！是 startup + (VLEN-1)。流水线只有第一个要等全部 latency
  • Scalar 不要忘 DECBNZ 的 2 cycles
  • 2 banks 时要想清楚哪些 LD/ST 能并行

13. Systolic Array — 脉动阵列

像心脏泵血一样，数据有节奏地"脉动"流过一排排 PE。每个 PE 做一次 MAC（乘加），然后把数据传给邻居。数据不回主存，在 PE 之间直接传，省带宽。

数据怎么流？

Weights

预加载到每个 PE 的寄存器，不动（Weight Stationary）

Activations

从左→右逐 cycle 流过每个 PE

Partial Sums

从上→下累加，最底下输出最终结果

2×2 Systolic Array 做矩阵乘

$\begin{bmatrix}a&b\\c&d\end{bmatrix} \times \begin{bmatrix}e&f\\g&h\end{bmatrix}$

T=0: 权重 [a,b,c,d] 预加载到 4 个 PE
T=1: PE(0,0) 收到 e，算 a×e（psum=ae）
T=2: PE(0,0) 收到 g，算 ae+b×g → 得到最终结果 ae+bg！
同时 PE(0,1) 收到 e（从左边传来），算 a×f
数据"波纹式"扩散，所有 PE 持续工作

Google TPU v1: 256×256 阵列 = 65536 MAC/cycle，8-bit int

14. Dataflow — WS / OS / IS

"Stationary" = 哪个数据不动，留在 PE 的寄存器里被反复使用。
核心思想：让被复用最多的数据留在离 PE 最近的地方（RF），减少搬数据。
Dataflow什么不动什么流动优化目标典型
Weight StationaryWeight 在 RFActivation 广播
Psum 累加传递		最大化 weight reuseTPU, NVDLA
Output StationaryPartial Sum 在 RFW 和 Act 流入减少 psum 写回次数一些 FPGA/ASIC
Input StationaryActivation 在 RFWeight 广播
Psum 传递		最大化 input reuse某些 CNN 加速器

卷积的 Loop Nest（理解就行）

$$O_{m,p,q} = \sum_{c=0}^{C-1}\sum_{r=0}^{R-1}\sum_{s=0}^{S-1} I_{c,\,p+r,\,q+s} \times F_{m,c,r,s}$$

选择不同 dataflow = 选择把哪层循环放最内层（最内层的变量变化最频繁 → 需要复用）

15. GPU & SIMT

程序员写的是一个 thread 的代码（SPMD），GPU 硬件自动把 32 个 thread 打包成一个 Warp，用 SIMD 方式执行。
这就是 SIMT：程序员以为自己写的是独立线程（MT），但硬件用 SIMD 跑。
术语是什么类比
SP (Streaming Processor)最小执行单元，1 个 ALU一个工人
SM (Streaming Multiprocessor)多个 SP + shared memory + register file一个车间
Warp32 个 thread 一起执行同一指令32 个工人做同一动作
Block程序员定义的线程组，映射到 SM一个项目组分到某车间
FGMTFine-Grained Multithreading：一个 warp 等内存时，切换到另一个 warp一组人等材料时让另一组干活

SIMD vs SIMT

SIMD（传统）
  • 程序员必须用向量指令（如 AVX）
  • 必须知道向量长度
  • Lock-step，所有 lane 强制同步
SIMT（GPU）
  • 程序员写标量代码（看起来是普通线程）
  • 不需要知道 warp 大小
  • Thread 可以 diverge（if/else），硬件自动处理

P1: Gradient Descent

$\hat{y}=2x_1+5x_2$, 数据点 $(x_1, x_2, y)=(3, 1, 15)$, $\alpha=0.05$, MSE。求一步更新后的 $w_1$ 和 $w_2$。

点击查看详细解答
1. 算预测: $\hat{y}=2(3)+5(1)=11$
2. 算误差: $\hat{y}-y=11-15=-4$
3. 算梯度:
$\frac{\partial L}{\partial w_1}=2(-4)(3)=-24$
$\frac{\partial L}{\partial w_2}=2(-4)(1)=-8$
4. 更新:
$w_1=2-0.05(-24)=2+1.2=\mathbf{3.2}$
$w_2=5-0.05(-8)=5+0.4=\mathbf{5.4}$
验证方向: 预测偏小，所以两个 w 都变大 ✓

P5: Memory Bank Conflict

有 16 banks。以下哪些 stride 会产生 bank conflict？为什么？
a) stride=1   b) stride=3   c) stride=4   d) stride=8   e) stride=16

点击查看详细解答
a) stride=1: GCD(1,16)=1 → 用满 16 个 bank → 无冲突 ✓
b) stride=3: GCD(3,16)=1 → 用满 16 个 bank → 无冲突 ✓
c) stride=4: GCD(4,16)=4 → 只用 16/4=4 个 bank → 有冲突 ✗
访问: bank 0,4,8,12,0,4,8,12...
d) stride=8: GCD(8,16)=8 → 只用 2 个 bank → 严重冲突 ✗
访问: bank 0,8,0,8...
e) stride=16: GCD(16,16)=16 → 只用 1 个 bank → 最严重! ✗
所有访问都打到 bank 0

P6: Vector Code 算周期

for(i=0..99) C[i] = A[i] * B[i]
LD latency=11, MUL latency=6, ST latency=11, 16 banks, stride=1

点击查看详细解答
代码:
MOVI VLEN=100 (1c), MOVI VSTR=1 (1c)
VLD V0=A (11+99=110c), VLD V1=B (110c)
VMUL V2=V0*V1 (6+99=105c), VST C=V2 (110c)
总计: 1+1+110+110+105+110 = 437 cycles
Scalar 对比 (1 bank): 4 + 100×(11+11+6+11+2) = 4+4100 = 4104 cycles

P7: 多通道 Conv + im2col

Input $6\times6\times3$, 4 个 $3\times3\times3$ filter, S=1, P=0
a) Output 大小？b) im2col 矩阵的维度？c) 总 FLOPs？

点击查看详细解答
a) Output: (6-3+0)/1+1 = 4, 4 个 filter → 4×4×4
b) im2col:
Filter flatten: 3×3×3 = 27 → 行向量 [1×27]，4 个 filter → Weight 矩阵 4×27
滑动窗口数: 4×4 = 16 个位置，每个窗口 27 元素 → im2col 矩阵 27×16
矩阵乘: [4×27] × [27×16] = [4×16] → reshape 成 4×4×4 ✓
c) FLOPs: 4 filters × 4×4 output × 3×3×3 kernel × 2 = 4×16×27×2 = 3456
（每个输出位置做 27 次 MAC = 27 乘 + 27 加 = 54 ops，4个filter × 16个位置 × 54）

P8: Systolic Array AI

一个 4×4 weight-stationary systolic array，每列加载 N 个 activation 流过。
FP32。求 activation 的 AI 和 weight 的 AI。

点击查看详细解答
每个 activation: 流过 4 个 PE，每 PE 做 4 次 MAC = 4×4×2 = 32 FLOPs
Activation AI:
FLOPs = 32N（N 个 activation 各 32 ops）
Bytes = 4列 × N × 4B = 16N
$AI_{act} = 32N/16N = \mathbf{2.0}$
Weight AI:
每个 weight 被 N 个 activation 复用 → FLOPs = 2N
Bytes = 4 (1 个 FP32)
$AI_{wt} = 2N/4 = \mathbf{0.5N}$
N 越大 weight 复用率越高!
