1. Gradient Descent 梯度下降

核心思想

迭代更新参数最小化 Loss。每步沿梯度负方向走一小步。

$$w_{new} = w_{old} - \alpha \cdot \frac{\partial L}{\partial w}$$

$\alpha$ = Learning Rate, $\frac{\partial L}{\partial w}$ = Gradient

算法步骤

  1. 初始化 $w$（Xavier/He）
  2. Forward：$\hat{y}=h(x)$
  3. Loss：$L=\frac{1}{N}\sum(\hat{y}_i-y_i)^2$
  4. Backward：Chain Rule 算 $\frac{\partial L}{\partial w_j}$
  5. Update：$w_j \leftarrow w_j - \alpha\frac{\partial L}{\partial w_j}$
  6. 重复直到收敛

GD 变体

Batch GD

全部数据算梯度。稳定但慢

SGD

1 个样本。快但噪声大

Mini-batch

一小批。平衡（最常用）

Learning Rate

α 太大

震荡/发散

α 太小

收敛极慢，卡局部最小

考点写出 update formula；区分 Batch/SGD/Mini-batch；LR 影响。

2. Loss Functions

回归

MSE

$$L=\frac{1}{N}\sum(\hat{y}_i-y_i)^2$$

惩罚大误差；可微

MAE

$$L=\frac{1}{N}\sum|\hat{y}_i-y_i|$$

对 outlier 鲁棒

分类

Cross-Entropy

$$L=-\sum_{c} y_c\log\hat{y}_c$$

预测概率越高 loss 越小

Logistic (Binary)

$$L=-[y\log\hat{y}+(1-y)\log(1-\hat{y})]$$

3. Chain Rule & Backprop

Chain Rule

复合函数导数 = 各层导数之积。从 Loss 反向传播梯度。

$$\frac{\partial L}{\partial w}=\frac{\partial L}{\partial \hat{y}}\cdot\frac{\partial \hat{y}}{\partial z}\cdot\frac{\partial z}{\partial w}$$

线性回归

$\hat{y}=w_1x_1+\cdots+w_nx_n$, MSE

$$\frac{\partial L}{\partial w_j}=\frac{2}{N}\sum(\hat{y}_i-y_i)\cdot x_j$$

Backpropagation

  1. Forward: 逐层算输出，存中间值
  2. Compute Loss
  3. Backward: Chain Rule 逐层算 $\frac{\partial L}{\partial W^{(l)}}$
  4. Update: $W^{(l)}\leftarrow W^{(l)}-\alpha\frac{\partial L}{\partial W^{(l)}}$

Vanishing Gradient

Sigmoid/Tanh 导数<1，多层相乘梯度衰减。

ReLU

正区间梯度=1

Weight Init

Xavier/He

Batch Norm

$\hat{x}=\frac{x-\mu}{\sqrt{\sigma^2+\epsilon}}$

6. Conv vs Cross-Correlation

Convolution

Flip kernel 再滑动

$$(f*g)(t)=\sum_\tau f(\tau)g(t-\tau)$$

交换律、结合律

Cross-Correlation (CNN)

不翻转直接滑动

$$(f\star g)(t)=\sum_\tau f(\tau)g(t+\tau)$$

kernel learned，翻不翻一样

考点CNN 用 cross-correlation 不翻转，但习惯叫 convolution。

8. Roofline Model

Arithmetic Intensity

$$AI=\frac{W}{Q}=\frac{\text{FLOPs}}{\text{Bytes}}$$

Roofline

$$\text{GFLOP/s}=\min\begin{cases}\text{Peak GFLOP/s}\\AI\times\text{Peak GB/s}\end{cases}$$

两区域

Memory Bound

$AI<$ Peak/BW. 受带宽限制

Compute Bound

$AI\geq$ Peak/BW. 已达峰值

Transition

$$AI_{trans}=\frac{\text{Peak GFLOP/s}}{\text{Peak GB/s}}$$

AI 对比

10. SIMD: Time vs Space

Array Proc（空间）
  • 多 PE 同时相同操作不同数据
  • Same op @ same time
  • 每 lane 有 ALU + RF
Vector Proc（时间）
  • 同一 PE deep pipeline 处理连续数据
  • Same op @ same space
  • CRAY-1 pipeline
区分Array=空间铺开多单元同时做; Vector=时间流水一单元连续做

12. Memory Banks

What

内存分多个 bank，各自独立访问，共享 bus。

Why

问题

单 memory latency 高（11cyc）

解决

banking 并行：一bank等待时其他服务

Advantages

  • 每cycle启动新bank→1 elem/cyc
  • 支持向量连续加载
  • bandwidth 匹配处理器

When Best

条件
  1. Stride=1 连续元素交错在不同bank
  2. Banks ≥ Latency（如lat=11需≥11 banks）
  3. → 维持 1 elem/cyc

CRAY-1

参数值
Banks16
Vec Regs8×64 elem×64bit
Bank Lat11 cyc (16>11✓)

Stride Problem

Stride 导致访问集中同一 bank → bank conflict → 性能骤降

13. Scalar vs Vector Code

for(i=0..49) C[i]=(A[i]+B[i])/2

Scalar (304 instr)
MOVI R0=50     ;1c
MOVA R1=A      ;1c
MOVA R2=B      ;1c
MOVA R3=C      ;1c
X: LD R4=[R1++] ;11c
   LD R5=[R2++] ;11c

2 banks: 4+50×30=1504c

Vector (7 instr)
MOVI VLEN=50  ;1c
MOVI VSTR=1   ;1c

VLD  V1=B     ;11+49=60

VSHR V3=V2>>1 ;1+49=50

16 banks: 285c

Speedup: 2004/285≈7×

Vector latency = startup+(VLEN-1). 16 banks>11 lat → 不stall
